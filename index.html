Configurar Ambiente de Desenvolvimento;

Configure o ambiente de desenvolvimento na sua máquina. Caso você já tenha as ferramentas abaixo,
instaladas e configuradas na sua máquina, desinstale a ferramenta quando estiver fazendo a respectiva tarefa,
e instale e configure novamente, do zero. Realize as seguintes tarefas, na ordem:

  1. Instale e configure o Apache 2.4;
  2. Instale e configure o PHP7.4;
  3. Instale e configure o MySQL 8+;
  4. Instale e configure o DBeaver;
    1. Configure também, uma conexão a uma base vazia;
  5. Instale e configure o Git oficial, de https://git-scm.com/
  6. Instale e configure o VSCode +;
    1. Configure pelo menos 4, podem ser mais, extensões realmente úteis, para PHP, HTML, CSS e Javascript;

Primeiro Passo: Instalação e Configuração do Apache2;

  1. Primeiro Pesquisei o Apache2 e o resultado me levou para;
    • https://www.apachelounge.com/download/

  2. Fazer o download e extrair o "Apache2" / Entrar no diretório;

  3. Pegar o diretório do "Apache24" e colocar no "(C:)";

  4. Procurar o diretório "Conf" / Abrir o arquivo "httpd.conf" com algum editor de texto;

  5. Em seguida: Ir no terminal do Windows e executar como administrador e verificar se a porta 80
    esta sendo utilizada com o comando netstat -anb | findstr 0.0.0.0:80

  6. Depois de verificar a porta, volte para o arquivo "httpd.conf" e
    execute o comando "Ctrl + f" (Barra de Pesquisa) para pesquisar "C:/Apache24" ou Define SRVROOT para saber a onde
    esta a raiz do servidoz, é necessário que o diretório esteja no mesmo caminho;

  7. Agora vem a parte dos comandos para executar as configurações do "httpd.conf";
    1. "Ctrl + f" - #ServerName www.example.com:80 / Descomentar e alterar para
      localhost:80
    2. Após isso abra o terminal e execute os seguintes comandos;
    3. cd c:\Apache24>cd bin
    4. c:\Apache24\bin>dir
    5. c:\Apache24\bin>httpd.exe -k install

  8. Depois de executar o comando e instalar, vá em "Serviços" do Windows e verifique se o "Apache2.4"
    esta la e inicie o serviço;

  9. Se iniciar, deu tudo certo na primeira etapa e o Apache foi instalado e configurado corretamente;

  10. "Ctrl + f" AddType application/x-gzip .gz .tgz / Apenas vai adicionar os seguintes comandos; - (finalizando as
    configurações restantes);
    1. AddHandler application/x-httpd-php .php
    2. AddType application/x-httpd-php .php .html
    3. Action application/x-httpd-php "C:\PHP7.4"\php-cgi.exe

  11. "Ctrl +f" LoadModule xm12enc_module modules/mod_xm12enc.so / Apenas vai adicionar o seguinte comando;
    1. LoadModule php_module "C:\PHP7.4\php7apache2_4.dll

  12. Ir no final do arquivo "httpd.conf" e adicionar o comando;
    1. PHPIniDir "C:\PHP7.4"
    2. Timeout 240

  13. "Ctrl + f" DirectoryIndex index.html / Alturar para: DirectoryIndex index.php index.html

  14. Após finalizar todas essas configurações, vá em "Serviços" e reiniciei o serviço do Apache2.4
    se caso não der error, instalações e configurações feitas com sucessos;

  15. E pronto, Instalação e Configuração feita com sucesso;

Segundo Passo: Instalação e Configuração do PHP7.4;

  1. Primeiro Pesquisei o PHP7.4 e o resultado me levou para;
    • https://windows.php.net/download#php-8.1

  2. Fazer download e extrair o "PHP 7.4" - VC15 x64 Thread Safe (zip);

  3. Pegar o diretório do "PHP7.4" e colocar no "(C:)";

  4. Renomear o diretório para "PHP7.4" (Opcional);

  5. Entrar no diretório e procurar o arquivo "php.ini-development"
    renomea-lo para "php.ini" / Abrir com algum editor de texto;

  6. Agora vem a parte dos comandos para executar as configurações do "php.ini";
    1. "Ctrl + f" - ;extension=curl, ;extension=gd2, ;extension=openssl,;extension=Pdo_mysq, ;extension=mbstring,
      Descomentar;
    2. "Ctrl + f" - ;session.save_path = "/temp" / Descomentar e alterar para: session.save_path = "C:\Windows\temp";
    3. "Ctrl +f" error_reporting = E_ALL / Alterar para: error_reporting = E_ALL & ~E_DEPRECATED & ~E_STRICT;
    4. "Ctrl + f" ;extension_dir = "ext" / Descomentar;

  7. Depois que terminar de configurar o arquivo "php.ini", volte para o arquivo "httpd.conf" e finalize as configurações
    restantes;

Terceiro Passo: Instalação e Configuração do MySQL 8+;

  1. Primeiro Pesquisei o MySQL e o resultado me levou para;
    • https://www.mysql.com/downloads/

  2. Procurar por "MySQL Community Edition (GPL)" / Fazer download do "MySQL on Windows (Installer & Tools)";
    1. "MySQL Installer (Servidor mysql + interface Workbench)";
    2. "MySQL Installer 8.0.13 - Windows (x86, 32-bit), MSI Installer - 8.1.0.13 (313.8M) Download";

  3. Assim que o download for concluído / Abrir o instalador;
    1. Aceitar os termos de licenças;
    2. Developer Default / Permite instalar todo o MySQL;
    3. C:\Programa Files\MySQL\MySQL Server 8.0 / Diretório de instalação;
    4. Upgrade dos pré-requistos do MySQL - MySQL para Visual Studio e Connector do Python;
    5. Após isso dê um "Execute" para dar continuação a toda instalação;
    6. Em seguida: Será necessário configurar o MySQL Server, MySQL Router, Samples e Examples;
    7. Standalone MySQL Server / Classic MySQL Replication (Padrão) / Nessa parte é melhor
      deixar a configuração recomendada;
    8. Config Type: Development Computer (Padrão) / Nessa parte é melhor
      deixar as configurações recomendadas;
    9. Use Strong Password Encryption For Authentication (Padrão) / Usar Senha forte para
      autenticar;
    10. Em seguida: Será necessário criar uma senha para autenticar o MySQL (Opcional);
    11. MySQL User Accounts / Após criar a senha, terá que adicionar um usuário no MySQL;
    12. Add User / Criar um usuário de autenticação (Opcional);
    13. Host: All Hosts (Padrão) / Ponto de acesso do usuário, para todos os lugares;
    14. Roles: DB Admin (Padrão) / Privilegio do usuário;
    15. Colocar a senha criada para finalizar a autenticação;
    16. Windows Service Name: MySQL80 (Padrão) / Nome do serviço - (Nessa parte é melhor
      deixar as configurações recomendadas);
    17. Após isso dê um "Execute" para dar continuação a toda instalação;
    18. O MySQL Router não precisa ser configurador, a não ser que seja um profissinal de
      de infraestrutura;
    19. Samples e Examples - MySQL Server8.0.13 (Padrão) / Bancos de exemplos
    20. User: root and Password / Usuário root (Padrão) ou pode ser o usário que criou
      e a senha precisa ser a que foi criada;
    21. Após isso dê um "Execute" para dar continuação a toda instalação;
    22. Instalação concluída / Nessa parte é melhor deixar as configurações recomendadas;

  4. Abrir o MySQL Workbench / Abrir o MySQL Connections, vai tentar conectar com o root e
    pôr a senha que foi criada;

  5. Caso não queira conectar com o root é só selecionar no MySQL Connections (+) e pôr o usuário que foi criado
    junto com a senha criada;

  6. Após abrir, vá em SQL (+) e execute o comando show databases; e dê "Ctrl + enter"; / irá executar
    e mostrar os bancos de dados;

  7. Execute também o comando use sakila; dê novamente "Ctrl + enter" / Em seguida: Dê o comando
    show tables; / irá mostrar as tabelas;

  8. Execute mais uma vez e dê mais um comando select * from actor / irá exibir todos os campos e todas as colunas
    da tabela actor;

  9. E Pronto, Instalação e Configuração Feita Com Sucesso;

Quinto Passo: Instalação e Configuração do Git oficial;

  1. Primeiro Pesquisei o Git oficial e o resultado me levou para;
    • https://git-scm.com/download/win

  2. Fazer download do "Standalone Installer (Git for Windows - 64-bit for Windows Setup)";

  3. Assim que o download for concluído / Abrir o instalador;
    1. Aceitar os termos de licenças;
    2. C:\Programa Files\Git / Diretório de instalação;
    3. Select Components / Nessa parte é melhor deixar as configurações recomendadas;
    4. Choosing the default editor user by git - which editor would you like Git to use / Escolha um
      editor de texto de sua preferência ou o que estar usando atualmente;
    5. Let Git decide (Padrão / Opcional) ou Overside the default branch same for new repositório (Opcional) /
      Branch como master (Padrão / Opcional) ou Branch como main, truck e development (Opcional);
    6. Adjusting your PATH environment - use Git from Git bash only, Git from the command line and also from
      3rd-party software (Padrão) e use Git and optional Unix tools from the Command Prompt / Usar o Git apenas
      com o Git bash, usar o terminal customizado do Git e Usar o Git junto com CMD; / Selecionei o usar o Git
      com"use Git and optional Unix tools from the Command Prompt...";
    7. Choosing the SSH executable - use bundled OpenSSH (Padrão) / usar o OpenSSH com o Git (Padrão);
    8. Depois disso o restante da instalação e configuração pode-se deixar como Padrão;
    9. Após isso a instalação e configuração será feita com sucesso;
    10. Depois vá na área de trabalho dentro de um diretório de preferência dentro do seu projeto atual / Com o
      botão direito abra o "Git bash here";

  4. Use os seguintes comandos;
    1. git config --global user.name "(seu nome)";
    2. git config --global user.email "(seu email do Github);
    3. git init / para dar inicio ao projeto;
    4. touch test.txt / para testar e subir pro projeto;
    5. git add . / informar que tem novos arquivos e novas modificações;
    6. git commit -m "Primeiro Commit" / informar que tem intenção de subir os arquivos;
    7. ssh-keygen -t rsa -C "(seu email do Github)"
      8. / Criar uma chave ssh para conectar com a conta do
      Github e subir os arquivos;
    8. Dê enter / Depois crie uma senha;
    9. Criar senha para ficar vinculada a sua chave ssh (A senha não precisa ser a mesma do Github;
    10. Após isso irá gerar uma chave ssh para poder transferir os arquivos;
    11. notepad (e o caminho do diretorio).ssh/id_rsa.pub / Pegar o texto do bloco de notas e colocar no Github;
    12. Depois vá no site do Github (com Sua conta) vá em configuração e procure SSH AND GPG KEYS /
      Coloque o texto copiado dentro do SSH KEYS, coloque o titulo "Minha SSH" também para facilitar e
      adicione;
    13. Após isso ssh conectado, para verificar se estar pegando, volte no diretório do projeto e abra o
      "Git bash here" novamente e execute o comando: ssh -T git@github.com / Irá perguntar se pretende
      continuar com a coneção, digite "yes" e pedira a senha criada para chave ssh / Caso dê certo: Autenticação
      concluída;
    14. Proximo passo é ir no Github e criar um repositório teste-ssh para o projeto (Opcional);
    15. Último passo é ir novamente no "Git bash here" e dar o comando git remote add origin git@github.com:Nome do
      Usuário do github/teste-ssh / Seu nome no github junto com o nome do repositório criado;
    16. Em seguida: Dê o comando: git push origin main / Para testar se estar funcionando;
    17. Caso dê certo: Autenticação concluída;

  5. E Pronto, Instalação e Configuração Feita Com Sucesso;
